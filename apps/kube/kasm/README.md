# kasm — Kubernetes Deployment

Browser-based Discord workspace running inside a [KASM](https://kasmweb.com) container. All egress flows through a [Gluetun](https://github.com/qdm12/gluetun) WireGuard tunnel — the workspace shares Gluetun's network namespace, so it has no direct internet path. If Gluetun drops, traffic stops instead of leaking through the cluster's public IP.

## Manifests

| File                                 | Purpose                                                                                                             |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------- |
| `application.yaml`                   | ArgoCD `Application` pointing at `apps/kube/kasm/manifest`; ignores `spec.replicas` so manual scale-ups don't drift |
| `manifest/namespace.yaml`            | `kasm` namespace                                                                                                    |
| `manifest/rbac.yaml`                 | `kasm-init-sa` SA + Role/RB to PATCH `Secret/kasm-vnc-pw`; cross-ns Role/RB letting `kbve-app` GET it               |
| `manifest/secret-vnc-pw.yaml`        | `kasm-vnc-pw` Secret stub (placeholder password; init container overwrites on every pod start)                      |
| `manifest/deployment.yaml`           | `kasm-vpn` Deployment (gluetun + workspace + 2 init containers) and `kasm-vpn-service`                              |
| `manifest/pvc.yaml`                  | 5Gi Longhorn PVC for `/home/kasm-user`                                                                              |
| `manifest/sealed-vpn-wireguard.yaml` | SealedSecret holding the WireGuard provider, type, and keys                                                         |
| `manifest/network-policy.yaml`       | CiliumNetworkPolicy locking egress to the VPN tunnel                                                                |

## Topology

| Container       | Image                                  | Role                                                         |
| --------------- | -------------------------------------- | ------------------------------------------------------------ |
| `fix-pvc-perms` | `busybox:1.37` (init)                  | `chown -R 1000:1000 /home/kasm-user`                         |
| `gen-vnc-pw`    | `curlimages/curl:8.10.1` (init)        | Generate 32-byte random password, PATCH `kasm-vnc-pw` Secret |
| `gluetun`       | `ghcr.io/qdm12/gluetun:v3.41.1`        | WireGuard tunnel + kill-switch + HTTP proxy on `8000`        |
| `workspace`     | `kasmweb/discord:1.18.0-rolling-daily` | KASM browser desktop on `6901` (shares gluetun netns)        |

`replicas: 0` by default. Scale to `1` manually when needed; ArgoCD's `ignoreDifferences` on `/spec/replicas` keeps the manual scale-up from reverting.

## Per-session VNC password rotation

Every pod start is treated as a new session. The VNC password is generated by the `gen-vnc-pw` init container before the workspace boots:

1. `gen-vnc-pw` runs as `kasm-init-sa`, generates `[A-Za-z0-9]{32}`, and PATCHes the `password` field of the `kasm-vnc-pw` Secret via the in-cluster kube API.
2. `workspace` consumes `VNC_PW` via `secretKeyRef` against `kasm-vnc-pw`. kubelet resolves the env at pod create time, after the init has already PATCHed the Secret.
3. The Deployment annotation is intentionally `secret.reloader.stakater.com/reload: "vpn-wireguard"` — **not** the generic `reloader.stakater.com/auto: "true"` — because `auto` would see the per-session VNC_PW PATCH and reroll the pod forever.

## Auto-login flow

`axum-kbve` exposes a JWT-gated launch endpoint that reads the rotated password from the cluster and 302s the user straight into the desktop:

```
User → kbve.com/dashboard/kasm/launch
  → axum-kbve: require_dashboard_view (Supabase JWT + DASHBOARD_VIEW perm)
  → axum-kbve: GET secrets/kasm-vnc-pw via in-cluster API (kbve-app SA)
  → 302 Location: /dashboard/kasm/proxy/?password=<value>&autoconnect=1&resize=remote
  → noVNC autoconnects → workspace
```

The user never sees the VNC password. Bookmarking `kbve.com/dashboard/kasm/launch` becomes the one-click entry point.

## Hardening

### Pod-level

- `fsGroup: 1000`
- `seccompProfile: RuntimeDefault` applied to every container
- Pod runs under `kasm-init-sa` so the gen-vnc-pw init can call the kube API. The SA's Role is scoped to a single Secret resourceName, so workspace + gluetun riding the same SA cannot exfiltrate other secrets.

### `fix-pvc-perms` (init)

- `runAsUser: 0`, `runAsNonRoot: false`
- `allowPrivilegeEscalation: false`, `readOnlyRootFilesystem: true`
- Capabilities: drop `ALL`; add `CHOWN`, `DAC_OVERRIDE`, `FOWNER`

### `gen-vnc-pw` (init)

- `runAsUser: 65532`, `runAsNonRoot: true`
- `allowPrivilegeEscalation: false`, `readOnlyRootFilesystem: true`
- Capabilities: drop `ALL`

### `gluetun`

- Image-default user (root). Runs the WireGuard handshake and manages the shared netns; pinning `runAsNonRoot` would break the tunnel.
- `allowPrivilegeEscalation: false`
- Capabilities: drop `ALL`; add `NET_ADMIN`

### `workspace`

- `runAsNonRoot: true`, UID/GID `1000` (matches `kasm-user` in KASM workspace images)
- `allowPrivilegeEscalation: false`
- Capabilities: drop `ALL`
- `readOnlyRootFilesystem` deferred — upstream image writes to `/tmp`, `/var`, `/home/kasm-user/.cache` during boot. Tracked as a follow-up.

## Known follow-ups

- `readOnlyRootFilesystem` on the `workspace` container — needs `emptyDir` mounts plus a live-test cycle.
- `gluetun` non-root via `PUID`/`PGID` — investigate whether the WireGuard handshake completes under the cluster's CNI when running gluetun as a non-zero UID.

## Egress lockdown

`network-policy.yaml` restricts egress from any pod with `app: kasm-vpn` to:

- kube-dns (UDP/TCP 53)
- in-cluster RFC1918 ranges (so the pod can reach the gluetun-managed tunnel itself)

External traffic only leaves the cluster through the WireGuard tunnel inside Gluetun. If the tunnel drops, gluetun's iptables kill-switch and Cilium policy together stop traffic instead of falling back to the node's IP.
