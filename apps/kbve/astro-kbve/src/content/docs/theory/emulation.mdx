<BentoProse id="ryujinx" eyebrow="Switch emulation" heading="Ryujinx">

Ryujinx was an open-source Nintendo Switch emulator written in C# on .NET, started in 2017 by the Brazilian developer gdkchan.
It was the first Switch emulator to boot commercial games in 2018 and grew into one of the most accurate emulators for the platform, with roughly 3,550 of about 4,300 tested titles rated playable by mid-2024.
Along the way it picked up a Vulkan graphics backend, a native macOS port for Apple Silicon (using the hypervisor for near-native ARMv8 execution), and a modern Avalonia-based interface.

On October 1, 2024, development ended abruptly.
Nintendo contacted gdkchan and offered an agreement to stop working on the project, and the GitHub organization, downloads, and Patreon were removed.
Unlike the Yuzu case earlier that year, there was no lawsuit or public settlement — the project simply went dark, with 1.1.1403 as the final official release.

The instructions that used to live on this page — the `choco install ryujinx` package, the official website downloads, and the RyuSAK setup companion — all target the discontinued original and are no longer maintained.
The community picked up the torch instead, and the most complete continuation is Ryubing, covered below.

</BentoProse>

<BentoProse id="ryubing" eyebrow="The continuation" heading="Ryubing">

[Ryubing](https://github.com/Ryubing) is the community fork that carries Ryujinx forward, described plainly on its GitHub organization as "a fork of the discontinued Switch emulator, Ryujinx."
It was started right after the October 2024 shutdown by GreemDev, a member of the original Ryujinx team, initially as a quality-of-life fork before being rebranded to Ryubing around the start of 2025.

Compared to where the original left off, Ryubing added:

- **Built-in LDN multiplayer** — local-wireless multiplayer over the internet, previously a separate community build, now integrated with a project-hosted LDN server.
- **Two release channels** — *Stable* for tested releases and *Canary* for automatic per-commit builds from the development branch, delivered through a built-in auto-updater.
- **Broader platform coverage** — Windows x64 and ARM64, Linux x64 and ARM64, and a universal macOS build, with a community-maintained Metal backend for Apple Silicon alongside the Vulkan and OpenGL ones.
- **Quality-of-life tooling** — an XCI trimmer, mod and cheat management inherited from Ryujinx, and ongoing performance and compatibility work (including partial SDK20 support so newer game updates like Tears of the Kingdom 1.4.x boot).

The project has had a bumpy legal ride.
In March 2025, Nintendo issued DMCA takedowns against Ryujinx forks on GitHub, which knocked Ryubing's main code repository offline; the source moved to a self-hosted Git at [git.ryujinx.app](https://git.ryujinx.app), while the issue tracker, compatibility list, and support repos stayed on GitHub.
After another DMCA wave in February 2026, GreemDev stepped away and the project shifted into maintenance mode under contributor LotP and the remaining team — infrastructure, compatibility tracking, and bugfix releases continue, but major feature development has slowed.

Related forks worth knowing about: **ryujinx-mirror** is a preservation-focused hard fork that only ships minor fixes, and **Kenji-NX** shares most of its changes with Ryubing while keeping closer to the original Ryujinx interface.

</BentoProse>

<BentoProse id="install" eyebrow="Get started" heading="Install">

Ryubing's downloads and documentation live at [ryujinx.app](https://ryujinx.app), with the setup guide at [docs.ryujinx.app](https://docs.ryujinx.app/guides/setup-guide/).
Pick a channel when you download:

- **Stable** — tested releases, the right default for most players.
- **Canary** — bleeding-edge builds generated from every commit; more fixes and features, more churn.

The general flow is the same on every platform:

1.  Download the build for your OS and architecture from the Ryubing website.
2.  Extract the archive to a location of your choice.
3.  Launch the emulator and install your dumped Switch firmware, then point it at your `prod.keys`.
4.  Add your game directory through the settings, select a title, and play.

The emulator does not ship firmware or keys — you dump both from your own hackable Switch using tools like Atmosphère and Lockpick_RCM, per the official setup guide.
Treat any third-party site bundling "Ryujinx keys" or firmware as unofficial and unsafe.

### Windows

Windows x64 and ARM64 builds are provided.
The old Chocolatey `ryujinx` package tracked the discontinued original and is stale, so grab the archive from the Ryubing site directly and keep it current through the built-in updater.
Keep your GPU drivers updated — Vulkan is the preferred backend.

### Linux

x64 and ARM64 archives are available, and community packaging exists (for example the AUR on Arch).
The same firmware and key steps apply.

### Mac

The macOS build is a universal binary and runs well on Apple Silicon, where the hypervisor executes ARMv8 code near-natively and a Metal backend is available.
This is a far better story than the original Ryujinx-era Mac experience previously noted on this page.

</BentoProse>

<BentoProse id="setup" eyebrow="Configure" heading="Setup">

RyuSAK, the setup companion we used to recommend here, targeted the original Ryujinx and is no longer maintained.
For a current install, the [official Ryubing setup guide](https://docs.ryujinx.app/guides/setup-guide/) covers everything post-install: firmware installation from an XCI or ZIP, key placement, and per-game configuration.
For compatibility questions, the project maintains a public [compatibility tracker](https://github.com/Ryubing) on GitHub and a Discord bot that answers lookups directly.

</BentoProse>
