### CI Main FanOut Pipelines

-   10:21PM

    Spent the day implementing a fanout approach to the workflows.
    The idea here is to move away from a sync based main and move towards an async deployment, which then creates issue tickets for failures.
    The end goal will be to have full loops with each code change then triggering an issue ticket, follow up response, ect... 
    Afterwards, I am thinking of integrating an additional layer of automation within the pipeline.

### Branch Retirement

-   01:45PM

    With the latest changes, it might be time to retire the alpha and beta branches and stick with just `main` and `dev` again.
    We can scale back the abstraction with the branches because we can use gitlab and azure to handle our alpha / beta.
    This would make things easier to build around and maintain while making the developement cycle easier to manage.

    The removed branches will be `alpha`, `beta` and `patch` workflows.
