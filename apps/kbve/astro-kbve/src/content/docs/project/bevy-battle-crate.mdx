    faq:
        - question: What does the bevy_battle crate do?
          answer: It provides generic turn-based battle types — damage resolution, effects, class procs, and enemy AI — as a pure Rust core plus an optional Bevy ECS plugin.
        - question: Can bevy_battle be used without Bevy?
          answer: Yes. The default "bevy" feature adds the BevyBattlePlugin and ECS derives, but disabling default features leaves a plain Rust type surface suitable for FFI cdylibs and headless serializers.
        - question: Are battles deterministic?
          answer: Yes. A seeded BattleRng resource drives combat rolls, so the same seed and inputs reproduce the same outcome for testing and replay.
