## Overview

`bevy_battle` is a generic turn-based battle plugin for [Bevy](https://bevyengine.org/) 0.18.
It packages **damage resolution, effects, class procs, and enemy AI** as a pure Rust
core with an **optional Bevy ECS layer**, so the same combat logic can run inside a
full game or inside a headless FFI build. Battles are driven by a **seeded RNG**
for deterministic, reproducible results.

## Features

- **Pure Rust core** — types, enums, and combat math in `types.rs` with zero Bevy
  dependency, usable from FFI cdylibs and mobile crates.
- **Optional Bevy ECS plugin** — the `bevy` feature (default) adds `Component` /
  `Event` / `Resource` / `Message` derives, the `BevyBattlePlugin`, snapshot/world
  helpers, and the full combat system pipeline.
- **Deterministic RNG** — `BattleRng` seeds every roll so battles reproduce exactly.
- **Global tuning** — `CombatModifiers` resource adjusts combat behavior in one place.
- **Enemy AI hook** — integrates `bevy_behavior` behavior trees for enemy turn logic.

## Usage

```rust
use bevy::prelude::*;
use bevy_battle::BevyBattlePlugin;

App::new()
    .add_plugins(BevyBattlePlugin)
    .run();
```

Disable the default `bevy` feature (`default-features = false`) to consume only the
pure type surface from FFI or headless crates.

## FAQ

**What does the bevy_battle crate do?**
It provides generic turn-based battle types — damage resolution, effects, class
procs, and enemy AI — as a pure Rust core plus an optional Bevy ECS plugin.

**Can bevy_battle be used without Bevy?**
Yes. The default `bevy` feature adds the `BevyBattlePlugin` and ECS derives, but
disabling default features leaves a plain Rust type surface suitable for FFI
cdylibs and headless serializers.

**Are battles deterministic?**
Yes. A seeded `BattleRng` resource drives combat rolls, so the same seed and
inputs reproduce the same outcome for testing and replay.
