## Overview

`bevy_items` provides proto-driven item definitions for [Bevy](https://bevyengine.org/)
games in the KBVE isometric engine. It compiles the shared `itemdb.proto`
schema into typed Rust structs via `prost` and wraps them in a searchable
`ItemDb` resource, so any game can load the same item registry and query it
consistently.

## Features

- **Proto-driven schema** — `itemdb.proto` is the single source of truth; `prost-build` generates the Rust item structs at build time.
- **Searchable `ItemDb` resource** — query items by slug, ULID, type flags, or rarity.
- **JSON loading** — `ItemDb::from_json()` parses `/api/itemdb.json`, handling string-to-enum conversion automatically.
- **Game-agnostic** — the same registry serves Isometric, DiscordSH, and other Bevy games.
- **Optional inventory adapter** — the `inventory` feature wires in `bevy_inventory`.

## Usage

```toml
[dependencies]
bevy_items = { path = "packages/rust/bevy/bevy_items" }
```

The item pipeline flows from `itemdb.proto` through `prost-build` into Rust
structs, while the same proto feeds a Zod schema and human-authored Astro MDX
content served as `/api/itemdb.json`, which `ItemDb::from_json()` loads at
runtime.

## FAQ

**What does the bevy_items crate do?**
It compiles the shared `itemdb.proto` schema into typed Rust structs via `prost`
and wraps them in a searchable `ItemDb` Bevy resource, so any game can load the
same item registry and query it by slug, ULID, type flags, or rarity.

**Where do bevy_items item definitions come from?**
The `itemdb.proto` file is the source of truth. Human-authored content lives in
Astro MDX files at `kbve.com/itemdb`, is served as `/api/itemdb.json` at build
time, and `ItemDb::from_json()` parses that JSON into proto structs with
automatic string-to-enum conversion.

**Is bevy_items tied to a specific game?**
No. It is game-agnostic — Isometric, DiscordSH, and other Bevy games load the
same proto item registry, keeping item data consistent across the KBVE
ecosystem.
