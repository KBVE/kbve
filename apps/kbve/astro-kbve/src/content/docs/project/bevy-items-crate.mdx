jsonld:
    faq:
        - question: What does the bevy_items crate do?
          answer: It compiles the shared itemdb.proto schema into typed Rust structs via prost and wraps them in a searchable ItemDb Bevy resource, so any game can load the same item registry and query it by slug, ULID, type flags, or rarity.
        - question: Where do bevy_items item definitions come from?
          answer: The itemdb.proto file is the source of truth. Human-authored content lives in Astro MDX files at kbve.com/itemdb, is served as /api/itemdb.json at build time, and ItemDb::from_json parses that JSON into proto structs with automatic string-to-enum conversion.
        - question: Is bevy_items tied to a specific game?
          answer: No. It is game-agnostic — Isometric, DiscordSH, and other Bevy games load the same proto item registry, keeping item data consistent across the KBVE ecosystem.
