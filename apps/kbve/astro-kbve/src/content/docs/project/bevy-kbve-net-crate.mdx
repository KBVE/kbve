jsonld:
    faq:
        - question: What does the bevy_kbve_net crate do?
          answer: It defines KBVE's shared lightyear multiplayer protocol for Bevy 0.18 — the replicated components, player inputs, message types, and channels used by both the client and the server so both halves speak the same wire format.
        - question: Which networking backend does bevy_kbve_net use?
          answer: It builds on lightyear 0.26 with replication, prediction, interpolation, native input, and netcode. The optional client feature adds WebSocket, WebTransport, and UDP transports, including a wasm32 WebSocket path for browser builds.
        - question: What optional features does the crate expose?
          answer: The npcdb feature pulls creature types from bevy_npc, the creatures feature adds capture and sync systems via bevy_tasker and dashmap, and the client feature enables transport selection and connection lifecycle for the client half.
