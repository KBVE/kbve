jsonld:
    faq:
        - question: What does the bevy_npc crate do?
          answer: It compiles the shared npcdb.proto into typed Rust structs via prost and wraps them in a searchable NpcDb Bevy resource, so any game can load one proto NPC registry and query it by slug, ULID, type flags, rarity, or creature family.
        - question: How do you load NPC data into bevy_npc?
          answer: Add BevyNpcPlugin to register an empty NpcDb resource, then populate it at startup from JSON with NpcDb::from_json, from a proto binary with NpcDb::from_bytes, or from a decoded proto with NpcDb::from_proto, and insert it via commands.
        - question: What is the optional creature feature?
          answer: The creature feature enables game-agnostic ECS components for creature pooling, capture, and interaction — CreaturePoolIndex, CreatureState, CapturedCreatures, CreatureCaptureEvent, and CreaturePlugin.
