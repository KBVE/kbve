## Overview

`bevy_player` is a kinematic character controller for [Bevy](https://bevyengine.org/) 0.18,
built on **Avian3D** physics. It handles **gravity, jumping, collision-aware
movement**, input-to-direction mapping, and **fall-damage events**, exposing a
single `PlayerPlugin` configured with a `PlayerConfig` so games get a working
character out of the box.

## Features

- **Avian3D physics** — 3D f32 kinematic body with collision-aware movement.
- **Configurable tuning** — `PlayerConfig` sets speed, gravity, and jump velocity.
- **ECS components** — `Player` and `PlayerPhysics` mark and drive the controller.
- **Direction mapping** — `DirectionMap` converts keyboard input into movement.
- **Fall damage** — `FallDamageEvent` fires on hard landings for the game to react.
- **Spawn helper** — `spawn_player_entity()` spawns a fully configured player.

## Usage

```rust
use bevy::prelude::*;
use bevy_player::{PlayerPlugin, PlayerConfig};

App::new()
    .add_plugins(PlayerPlugin::new(PlayerConfig::default()))
    .run();
```

## FAQ

**What does the bevy_player crate do?**
It is a kinematic character controller for Bevy 0.18 built on Avian3D physics,
providing gravity, jumping, collision-aware movement, and fall damage.

**How do I configure movement speed and jump?**
Pass a `PlayerConfig` to `PlayerPlugin::new` — it tunes speed, gravity, and jump
velocity, and `spawn_player_entity` spawns a fully configured player.

**How is fall damage handled?**
The controller emits a `FallDamageEvent` on hard landings so the game can apply
damage or feedback in its own systems.
