## Overview

`bevy_statemachine` is a thread-safe state snapshot bridge for
[Bevy](https://bevyengine.org/). It exposes ECS resources to external consumers —
Tauri IPC, WASM JavaScript, and lightyear networking — so a running game's state
can be read from outside the ECS world without racing the schedule. Snapshots are
available as JSON or bincode.

## Features

- **Thread-safe snapshot bridge** — exposes ECS state to external threads and
  consumers without blocking the Bevy schedule.
- **Multiple consumers** — designed for Tauri IPC, WASM JavaScript, and lightyear
  networking off the same snapshot.
- **Monotonic versioning** — `snapshot_version::<T>()` returns a monotonic counter
  so consumers can detect fresh state cheaply.
- **Dual serialization** — JSON via `serde_json` (default `serde` feature) and
  compact binary via the optional `bincode` feature.

## Usage

```rust
use bevy::prelude::*;

    App::new()
        .add_plugins(StateSnapshotPlugin::<MyState>::new())
        .run();
```

## FAQ

**What does the bevy_statemachine crate do?**
It is a thread-safe state snapshot bridge for Bevy that exposes ECS resources to
external consumers such as Tauri IPC, WASM JavaScript, and lightyear networking,
with both JSON and bincode serialization.

**How do you read a state snapshot from bevy_statemachine?**
Add `StateSnapshotPlugin` generic over your state type, then call
`get_snapshot::<T>()` for the latest snapshot, `get_snapshot_json::<T>()` for a
JSON string, or `snapshot_version::<T>()` for the monotonic version counter.

**What serialization formats does bevy_statemachine support?**
The default `serde` feature provides JSON serialization via `serde_json`, and the
optional `bincode` feature adds binary serialization for compact transport.
