        - question: What is the KBVE API?
          answer: The KBVE API is the central multi-tenant API for KBVE projects, implemented as the axum-kbve Rust service. It exposes shared endpoints for auth and data and is built on Supabase, with reference documentation aimed at onboarding new developers.
        - question: What technologies power the KBVE API?
          answer: The API is written in Rust (axum-kbve) with Python tooling and a SQL database, and it builds on Supabase, Strapi, and Appwrite as reference frameworks for the multi-tenant surface.
        - question: How is the KBVE API deployed?
          answer: It runs in production in the kbve namespace as the kbve/kbve image, deployed via apps/kube/kbve/manifest/kbve-deployment.yaml, with an axum-kbve-e2e test suite gating releases.
