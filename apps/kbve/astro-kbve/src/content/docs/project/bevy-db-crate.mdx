## Overview

`bevy_db` is a cross-platform async key-value persistence crate for
[Bevy](https://bevyengine.org/), built for the KBVE isometric engine. It exposes
one Bevy-facing API backed by two storage engines and keeps every read and write
off the game thread so the schedule never stalls.

- **Native** — `redb`, a pure-Rust B+tree store on disk.
- **WASM** — `rexie` over IndexedDB in the browser.

All I/O is dispatched off the game thread via
[`bevy_tasker`](/project/bevy-tasker-crate/). Game systems poll `DbRequest<T>`
handles with `try_recv()` to drain results without blocking the frame.

## Features

- **One API, two backends** — the same `Db` handle works on native and WASM; the
  backend is selected at compile time.
- **Off-thread I/O** — all persistence work is dispatched through `bevy_tasker`,
  so game systems never block on disk or IndexedDB.
- **Poll-based results** — pending operations return a `DbRequest<T>` that systems
  drain with `try_recv()`, fitting naturally into a Bevy schedule.
- **Backend-agnostic errors** — a single `DbError` enum abstracts over redb and
  IndexedDB failure modes.

## Usage

```rust
use bevy::prelude::*;
use bevy_db::{BevyDbPlugin, Db, DbRequest};

App::new()
    .add_plugins(BevyDbPlugin::default())
    .run();
```

## FAQ

**What does the bevy_db crate do?**
It provides async key-value persistence for Bevy with a single game-facing API and
two backends — `redb`, a pure-Rust B+tree, on native platforms and `rexie` over
IndexedDB in the browser on WASM.

**How does bevy_db avoid stalling the game loop?**
All disk and IndexedDB I/O runs off the game thread through the `bevy_tasker`
crate. Game systems poll `DbRequest` handles with `try_recv()` to drain results
without blocking the schedule.

**What is the public API surface of bevy_db?**
`BevyDbPlugin` initializes the backend and inserts the `Db` resource, `Db` is the
request handle for `get` / `put` / `delete`, `DbRequest<T>` is the pending async
result polled with `try_recv()`, and `DbError` is the backend-agnostic error enum.
