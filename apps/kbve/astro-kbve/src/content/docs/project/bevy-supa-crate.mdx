## Overview

Transport-agnostic Supabase / PostgREST client. The default feature set ships the `native` HTTP transport (`reqwest` + rustls). Bevy integration is opt-in so that lean consumers (JNI cdylibs, CLIs) stay slim.

## Features

| Feature | Description |
| --- | --- |
| `native` (default) | `reqwest` + rustls native HTTP transport |
| `wasm` | Browser `fetch` transport (stub — not yet implemented) |
| `bevy` | `BevySupaPlugin` + `SupaClient` Bevy `Resource` |

Disable default features when only the type surface is needed (build scripts, codegen).
