## Overview

`kbve` is the KBVE core Rust crate — an [Axum](https://github.com/tokio-rs/axum)
and [Diesel](https://diesel.rs/) boilerplate that provides shared building
blocks for producing backend APIs and micro-services. It bundles authentication,
markdown, wallet, referral, and gate modules behind Cargo feature flags so each
service links only what it needs.

## Features

- **Axum + Diesel base** — HTTP routing plus async Diesel (diesel-async, bb8 pool) for backend micro-services.
- **Authentication** — Argon2 password hashing and JWT (`jsonwebtoken`) handling.
- **`supabase` / `kv`** — Supabase integration and a Valkey-backed key-value layer via `jedi`.
- **`gate` / `gate-auth`** — an auth-proxy WebSocket surface (the kbve-gate sidecar) fronting services like n8n, Grafana, and Studio.
- **`image-gen`, `wallet`, `referral`, `markdown`, `lot`** — additional optional modules for the KBVE platform.
- **`legacy-sync-db`** — the older synchronous Diesel modules (guild, models, runes, schema, spellbook).

## FAQ

**What is the kbve crate?**
It is the KBVE core Rust crate — an Axum and Diesel boilerplate that provides
shared building blocks for producing backend APIs and micro-services across the
KBVE ecosystem.

**What features does the kbve crate expose?**
Cargo feature flags gate optional capabilities including `supabase`, `kv`
(Valkey-backed), `image-gen`, `wallet`, `gate-auth`, and `gate` (an auth-proxy
WebSocket surface), plus a `legacy-sync-db` path for the older Diesel
synchronous modules.

**What is the kbve gate feature for?**
The `gate` and `gate-auth` features provide an authentication-proxy surface
built on Axum WebSockets and reqwest streaming, used as the kbve-gate sidecar
that fronts services such as n8n, Grafana, and Studio.
