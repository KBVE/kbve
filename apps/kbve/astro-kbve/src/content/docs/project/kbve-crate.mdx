        - question: What is the kbve crate?
          answer: It is the KBVE core Rust crate — an Axum and Diesel boilerplate that provides shared building blocks for producing backend APIs and micro-services across the KBVE ecosystem.
        - question: What features does the kbve crate expose?
          answer: Cargo feature flags gate optional capabilities including supabase, kv (Valkey-backed), image-gen, wallet, gate-auth, and gate (an auth-proxy WebSocket surface), plus a legacy-sync-db path for the older Diesel synchronous modules.
        - question: What is the kbve gate feature for?
          answer: The gate and gate-auth features provide an authentication-proxy surface built on Axum WebSockets and reqwest streaming, used as the kbve-gate sidecar that fronts services such as n8n, Grafana, and Studio.
