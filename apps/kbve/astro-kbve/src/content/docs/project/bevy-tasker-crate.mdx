jsonld:
    faq:
        - question: What does the bevy_tasker crate do?
          answer: It provides cross-platform async task spawning for Bevy games. On desktop it runs tasks on a thread pool sized to available parallelism; on WASM it dispatches work to the browser microtask queue or web workers, all behind one API.
        - question: How does bevy_tasker behave on WASM versus desktop?
          answer: On WASM without atomics tasks run via queueMicrotask; with atomics, spawn dispatches Send futures to web workers via a shared work queue and spawn_local pins non-Send futures to the spawning thread. On desktop tasks run on a crossbeam-channel-backed thread pool.
        - question: Is the bevy_tasker API the same across platforms?
          answer: Yes. Every path returns an async_task::Task, so calling code is identical across targets — await it, drop it to detach, or cancel it.
