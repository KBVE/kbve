jsonld:
    faq:
        - question: What does the bevy_mapdb crate do?
          answer: It compiles mapdb.proto into typed Rust structs via prost and wraps them in a searchable MapDb Bevy resource, so any game can query zones, regions, and world object definitions by slug, ULID, biome, or type.
        - question: How do you load map data into bevy_mapdb?
          answer: MapDb parses from three sources — MapDb::from_json for the mapdb JSON artifact, MapDb::from_bytes for a proto binary, and MapDb::from_proto for an in-memory MapRegistry. Insert the result as a Bevy resource during startup.
        - question: Is bevy_mapdb tied to a specific game?
          answer: No. It is game-agnostic. Any Bevy game can load the same shared proto map registry; BevyMapDbPlugin registers an empty MapDb that the game populates at startup.
