    faq:
        - question: What is the bevy_behavior crate?
          answer: bevy_behavior is a game-agnostic behavior tree engine for Bevy. It provides the core BehaviorNode trait, Selector and Sequence composite nodes, a tick-based cooldown system, and observation traits so the same tree logic can drive NPCs across Minecraft, Bevy Isometric, Discord MUD, and Unity.
        - question: How is a behavior tree parameterized?
          answer: The tree is generic over an Observation type (an immutable snapshot the NPC can see, implementing Positioned, Healthed, Aware, and Ticked) and an Action type (a game-defined command enum). A mutable BehaviorContext carries the current tick and cooldown handles per evaluation.
        - question: Does bevy_behavior require Bevy?
          answer: No. The core crate is pure Rust with zero framework dependency. Enabling the optional bevy feature adds Bevy Resource and Component derives for in-ECS tree evaluation.
