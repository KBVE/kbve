# Graphify Data Storage

Centralized storage for semantic knowledge graphs generated by Graphify.

## Directory Structure

```
graphify/
├── output/              # Generated graph data (git-ignored)
│   ├── monorepo/       # Full monorepo graph
│   ├── apps/           # Per-app graphs
│   └── packages/       # Per-package graphs
├── configs/            # Graphify configuration
│   └── .graphify.yml   # Main config file
└── scripts/            # Build automation scripts
    └── build-monorepo-graph.sh
```

## Quick Start

### 1. Install Graphify CLI

```bash
# Global installation (recommended for development)
uv tool install graphifyy

# Or via the Python package
pnpm nx run graphify-wrapper:install
```

### 2. Build Full Monorepo Graph

```bash
# Using the wrapper package (recommended)
pnpm nx run graphify-wrapper:build-monorepo

# Or using the shell script directly
./packages/data/graphify/scripts/build-monorepo-graph.sh

# Or using Graphify CLI directly
cd packages/data/graphify
graphify ../../../ --output output/monorepo --config configs/.graphify.yml
```

### 3. Build App-Specific Graph

```bash
pnpm nx run graphify-wrapper:build-app --app=herbmail
```

### 4. Query the Graph

```bash
pnpm nx run graphify-wrapper:query --q="Where is user authentication handled?"
```

### 5. Export for Web

```bash
pnpm nx run graphify-wrapper:export-web
```

## Configuration

Edit `configs/.graphify.yml` to customize:

- **LLM Backend**: Switch between Ollama (local, free) and Anthropic Claude (API, paid)
- **Ignore Patterns**: Exclude specific files/directories
- **Analysis Mode**: AST-only (fast) vs semantic (thorough)

## Output Files

After building, each graph contains:

- `graph.json` - Full graph data (nodes, edges, communities)
- `graph.html` - Interactive visualization (standalone)
- `GRAPH_REPORT.md` - AI-generated insights and statistics

## API Access

Graphs are served via Astro API endpoints:

- **Full Monorepo**: `GET /api/graphify/monorepo.json`
- **Specific App**: `GET /api/graphify/apps/{app}.json`

## CI/CD Integration

Graphs are rebuilt daily via GitHub Actions (see `.github/workflows/ci-daily-graph.yml`).

## Storage Size

Typical sizes:

- Full monorepo: ~5-10 MB
- Single app: ~500 KB - 2 MB

Graph data is git-ignored to save repository space. Run builds locally or rely on CI.

## Troubleshooting

**Error: 'graphify' command not found**

```bash
uv tool install graphifyy
```

**Graph file too large**

```bash
# Use Git LFS for files >10MB
git lfs track "packages/data/graphify/output/**/*.json"
```

**Build takes too long**

```bash
# Use AST-only mode (no semantic analysis)
pnpm nx run graphify-wrapper:build-monorepo  # Default: no semantic

# Or enable semantic for deeper analysis
./packages/data/graphify/scripts/build-monorepo-graph.sh --semantic
```
