# kbve.nx Security + Graph Routes (capability-tagged, staged)

**Date:** 2026-07-18
**Status:** Approved (pending spec review)
**Builds on:** `2026-07-18-kbve-nx-daily-content-router-design.md`

## Problem

The security and graph dashboard MDX are generated by standalone `scripts/nx-*-to-mdx.py` from `ci-dashboard.yml`. The parse/render logic is already ported into `kbve.nx` (`parse_all_ecosystems`→`render_security_*`, `parse_graph`→`render_graph_mdx`, `alerts.fetch_all`) but no routes exist. Migrate them into the route registry so the daily **router → builder** pipeline handles them, and so future dashboards (report, kanban, proto, …) migrate the same way with **no workflow edits**.

## Design principle — scale to N routes, 2 jobs, 0 workflow edits

Every content job is a Python route. Adding a route = a new file in `kbve/nx/routes/` + register. The workflow is **data-driven by capability tags**, never by route name. Only a genuinely new toolchain (rare) adds one guarded setup step.

## Staged rollout (this PR = stage 1 only)

1. **Stage 1 (this PR):** build `security` + `graph` routes with output **parity** to the current scripts; framework changes; wire into `ci-daily-content.yml` (data-driven setup + manual dispatch). Register both as cadence **`on-demand`** so they do NOT run in the nightly `daily` matrix — avoids conflicting with `ci-dashboard.yml`, which keeps generating the same files until stage 3. Delete nothing yet.
2. **Stage 2 (validation, no code):** `workflow_dispatch` each route through router+builder; confirm the produced `security.mdx`/`graph.mdx` + JSON match today's dashboard output.
3. **Stage 3 (follow-up PR):** flip security/graph cadence → `daily`; remove the security + graph jobs from `ci-dashboard.yml`; delete `scripts/nx-security-to-mdx.py`, `scripts/nx-graph-to-mdx.py`, `scripts/fetch-github-alerts.py`.

## Framework changes (`kbve/nx/`)

**`router.py`** — `Route` gains `needs: tuple[str, ...] = ()` (capability tags: `node`, `rust`, `python`, `token`). `@route(name, cadence, needs=...)`. `select(cadence)` unchanged. Add `get_many`/allow router CLI to select by explicit route name too.

**`builder.py`** — `BuildContext` extended:
- `content_root: Path` (existing — `src/content/docs`)
- `public_dir: Path` (new — `apps/kbve/astro-kbve/public/data/nx`, resolved relative to content_root)
- `workdir: Path` (new — temp scratch for acquired raw JSON)
- `timestamp: str` (new — ISO 8601; injected by CLI, defaults to route-computed)
- `dry_run`, `inputs`, `date` (existing)
`Builder.plan_all(cadence)` and `build_one(route)` unchanged in shape; `build_one` passes the richer context.

**`plan` semantics by route kind:**
- Scaffold (journal): cheap file detect (existing).
- Generated (security/graph): `needs_work=True` always — regeneration is cheap-ish and the workflow's `git diff --cached --quiet` guard drops no-op regenerations, so no empty PRs. `plan` does NOT run acquisition (keeps the router job light).

**`cli.py`:**
- `router_main`: add `--route <name>` (repeatable) to emit an explicit matrix bypassing cadence (for manual dispatch); keep `--cadence`. Matrix entries now include `needs`: `{"include":[{"route":"security","needs":"node,rust,python,token"}]}` (comma string — easy `contains()` in Actions). Still writes `matrix`/`has_work` to `$GITHUB_OUTPUT`.
- `build_main`: add `--timestamp`, `--workdir`, `--public-dir` passthroughs to `BuildContext`.

## Routes

**`routes/security.py`** — `@route("security", cadence="on-demand", needs=("node","rust","python","token"))`.
`build(ctx)`:
1. Acquire into `ctx.workdir` (subprocess, tolerant fallbacks mirroring ci-dashboard):
   - `pnpm audit --json` → `npm`
   - `cargo audit --json` → `cargo`
   - `pip-audit --format=json` over `packages/python` reqs → `python`
   - `alerts.fetch_all(endpoint="code-scanning", token=env GITHUB_TOKEN)` → `codeql`
   - `alerts.fetch_all(endpoint="dependabot", token=…)` → `dependabot`
   - merge → raw dict.
2. `parse_all_ecosystems(raw)` → data.
3. `render_security_json(data)` → `public_dir/nx-security.json`; `render_security_mdx(data, ctx.timestamp)` → `content_root/dashboard/security.mdx`.
Return changed paths. Acquisition failures degrade to the same empty fallbacks the CI job uses (never hard-fail on a single feed).

**`routes/graph.py`** — `@route("graph", cadence="on-demand", needs=("node",))`.
`build(ctx)`:
1. `pnpm nx graph --file=<workdir>/nx-graph.json --open=false` (cwd = repo root).
2. `parse_graph(path)` → GraphData; `render_graph_mdx(graph, ctx.timestamp)` → `content_root/dashboard/graph.mdx`.
3. Copy raw graph JSON → `public_dir/nx-graph.json`.
Return changed paths.

Both reuse the existing render/parse functions verbatim → parity with the standalone scripts by construction.

## Workflow — `ci-daily-content.yml` (extend, still two jobs)

`workflow_dispatch` gains inputs:
- `route` (string, optional) — build exactly this route regardless of cadence.
- `cadence` (string, default `daily`).

**router job:** `kbve-nx-router` with `--route ${{ inputs.route }}` when set, else `--cadence ${{ inputs.cadence || 'daily' }}`. Emits `matrix` (incl. `needs`) + `has_work`.

**builder job** (matrix over routes) — **data-driven setup, gated on `matrix.needs`:**
```
- checkout dev
- setup python + uv + uv sync            (always)
- if contains(matrix.needs,'node'):  setup node 24 + pnpm 10.33.0 + pnpm install --frozen-lockfile
- if contains(matrix.needs,'rust'):  rustup stable + cargo install cargo-audit
- if contains(matrix.needs,'python'): pip install pip-audit
- run: uv run kbve-nx-build ${{ matrix.route }} --timestamp <ISO> --workdir <tmp> --public-dir apps/kbve/astro-kbve/public/data/nx
  env: GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}   (harmless when unused)
- git add apps/kbve/astro-kbve/src/content/docs apps/kbve/astro-kbve/public/data/nx
- git diff --cached --quiet && exit 0     (guard — drops no-op regenerations)
- branch auto/daily-<route>-<run-date>; commit '[skip ci]'; push; gh pr create --base dev --label auto-pr
```
Journal (`needs:""`) skips every heavy step → unchanged fast path. New routes reusing these tags need **zero** workflow change.
